# AI OPS, PART 6: DEFECT TAXONOMY FOR SUPPORT AI

**Date:** February 9, 2026
**Category:** 1. AI in Customer Support
**Recency Score:** 0-6 months (full weight)

## Key Takeaways
- Systematize AI failures by converting incident data into labeled defect categories
- Assign ownership and remediation paths to each defect type
- Create actionable feedback loops from AI errors back to model improvement

## Frameworks / Models
**Defect Taxonomy Structure:**
- Categorize AI support failures (e.g., hallucinations, routing errors, contextual misunderstanding, knowledge base gaps)
- Label each defect with: failure type, severity, root cause classification, owner accountability
- Map defects to fix paths (retraining, prompt adjustment, knowledge base update, escalation protocol)

## Implementation Steps
1. Collect AI failure incidents from production support interactions
2. Classify failures into consistent taxonomy buckets
3. Assign defect ownership (product, ops, data science, support leadership)
4. Define resolution path for each defect category
5. Close loop: track remediation time and prevent recurrence

## Guardrails & Anti-patterns
- **Anti-pattern:** Treating all AI errors as equivalent—requires granular classification to drive different fixes
- **Guardrail:** Ensure ownership prevents defect accountability gaps between teams
- **Guardrail:** Fix paths must be observable/measurable to validate improvement
